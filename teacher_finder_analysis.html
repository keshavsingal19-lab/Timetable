🔍 Teacher Finder — Analysis & Implementation Plan

Goal: Admin button "Update Teacher Finder" scrapes the SRCC timetable website, extracts which teacher is in which room at which time slot on which day, and sends that data to the Timetable app's Teacher Finder feature.

2. What Needs To Be Done

Key Insight: The existing sync_rooms flow already does 90% of what "Update Teacher Finder" needs. The real work is: (A) fix the parser to extract teacher IDs accurately, (B) extract the teacher legend for code→name mapping, and (C) add a clear "Update Teacher Finder" button in the Admin UI.
#TaskEffortFiles
A Fix Teacher ID Parser — The regex /^[A-Za-z]{1,4}$/ rejects codes containing digits. The SRCC system uses alphanumeric codes (C17, EG2, C4, EVSG1, VAC5, VAC25). Need to update to /^[A-Z]{1,5}\d{0,2}$/i and filter out known false positives (subject abbreviations like SEM, BCH, CLAW, HRM, etc). Medium Admin/sync_rooms.ts, Timetable/sync_rooms.ts
B Extract Teacher Legend — Parse the <b>CODE</b>-FULL NAME block at the bottom of each room page. Accumulate a global code→name map across all rooms. Send this map to the Timetable app so the Teacher Finder can display real names even for teachers not in the local DB. Medium Admin/sync_rooms.ts, new webhook endpoint or extend save_rooms
C Add "Update Teacher Finder" Button — In the Admin Dashboard, add a button that triggers the same sync_rooms flow. Could be: (1) a separate button that reuses the same API, or (2) simply rename/reposition the existing "Sync Room Data" button since it already does this. Low Admin/src/pages/Dashboard.tsx
D Ensure Teachers Are Synced to Timetable — The Teacher Finder UI reads from liveTeachers (loaded from /api/teachers). If a teacher code from the scraper doesn't exist in the Timetable's teachers table, they won't show up in the finder. The legend parsing (Task B) can populate missing teachers automatically. Medium Timetable/webhook/save_rooms.js or new webhook
E False-Positive Filter — Current parser extracts subject codes (CLAW, HRM, COST, etc.) as teacher IDs. Need a blocklist of known non-teacher codes or a smarter extraction strategy that uses the legend as ground truth. Medium Admin/sync_rooms.ts

3. Implementation Plan

Phase 1: Fix the Parser (Admin)

Task A + B + E — Improve Data Extraction

File: Admin/functions/api/sync_rooms.ts

  1. Add legend parser: New function parseTeacherLegend(html) that extracts the <b>CODE</b>-FULL NAME block → returns Map<string, string> of code→name.
  2. Fix extractTeacherId(): Update regex to /^[A-Z]{1,5}\d{0,2}$/i to allow alphanumeric codes.
  3. Add subject blocklist: Filter out known non-teacher segments: ['SEM', 'BCH', 'BAHE', 'JOINT', 'CLAW', 'HRM', 'COST', 'EVS', 'IB', 'BA', 'CA', 'ADVT', 'PME', 'TGNLC', 'TSH', 'InterMME']
  4. Cross-validate with legend: After extracting a potential teacher ID from a cell, check if it exists in the legend. If it does → confirmed teacher. If not → likely a subject code, skip it.
  5. Accumulate global teacher map: Merge legends from all rooms into a single teacherDirectory map.
  6. Include in webhook payload: Add teacherDirectory alongside rooms in the broadcast to Timetable.
// NEW: Parse teacher legend from bottom of page
function parseTeacherLegend(html: string): Map<string, string> {
  const legend = new Map<string, string>();
  // Match: <b>CODE</B>-FULL NAME,
  const regex = /<b>([A-Z0-9]+)<\/b>\s*-\s*([^,<]+)/gi;
  let match;
  while ((match = regex.exec(html)) !== null) {
    legend.set(match[1].toUpperCase(), match[2].trim());
  }
  return legend;
}

// FIXED: Allow alphanumeric codes
function extractTeacherId(segment: string, legend: Map): string | null {
  const BLOCKLIST = new Set(['SEM','BCH','BAHE','JOINT','CLAW',
    'HRM','COST','EVS','IB','BA','CA','ADVT','PME','TGNLC',
    'TSH','AEC','VAC','SEC','GE','DSE','II','III','IV','VI']);
  const parts = segment.split('-').map(p => p.trim());
  for (let i = parts.length - 1; i >= 0; i--) {
    const p = parts[i].toUpperCase();
    if (/^[A-Z]{1,5}\d{0,2}$/.test(p) 
        && !BLOCKLIST.has(p) 
        && legend.has(p)) {
      return p;
    }
  }
  return null;
}

Phase 2: Extend Webhook to Carry Teacher Directory

Task D — Sync Teacher Names to Timetable

Files: Admin/sync_rooms.ts, Timetable/webhook/save_rooms.js

  1. Extend broadcast payload: Add teacherDirectory: { [code]: fullName } to the webhook.
  2. Extend receiver: In save_rooms.js, if payload.teacherDirectory exists, upsert each entry into the teachers table (so Teacher Finder has names for all scraped codes).
// In Admin sync_rooms.ts — broadcast payload:
const payload = { 
  action: "save_rooms", 
  rooms: results,
  teacherDirectory: Object.fromEntries(globalTeacherMap)
};

// In Timetable webhook/save_rooms.js — receiver:
if (payload.teacherDirectory) {
  for (const [code, name] of Object.entries(payload.teacherDirectory)) {
    await env.DB.prepare(`
      INSERT INTO teachers (id, name, department) 
      VALUES (?, ?, 'SRCC')
      ON CONFLICT(id) DO UPDATE SET name=excluded.name
    `).bind(code, name).run();
  }
}

6. Parser Bug Examples

From real room R1 HTML, the current parser produces these false results:

Cell ContentCurrent OutputCorrect OutputIssue
Commerce-CLAW-RNC ["RNC"] ✅ ["RNC"] Works — pure alpha, 3 chars
Commerce-CA-C17 ["CA"] ❌ ["C17"] C17 rejected (has digit), CA is a subject
Economics-InterMME-EG2 [] ❌ ["EG2"] EG2 rejected (has digit), InterMME too long
EVS-EVS-II-FYV-AEC-4 ["AEC"] ❌ ["FYV"] AEC is a course code, FYV is the teacher
Commerce-TSH-C4-VAC25 ["TSH"] ❌ ["C4"] or ["VAC25"] TSH is a subject. C4/VAC25 rejected (digits)
Solution: The legend at the bottom of the HTML page is the ground truth. By parsing it first and using it as a whitelist, we eliminate all false positives. The legend for room R1 lists: AOK, ATS, AAG, RUA, VIS, EG2, RNC, C17, RKS, YAA, AKY, PAA, SJJ, PAC, NAD, ALK, JTW, FYV, C4, EVSG1.
